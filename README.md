# Sprint 10 KOH 26

Android-приложение с экраном чата, реализованное на Kotlin. Проект демонстрирует работу с `RecyclerView`, несколькими типами элементов списка, `DiffUtil`, обработкой ввода сообщения и базовой логикой обновления интерфейса. :contentReference[oaicite:0]{index=0}

## Описание

Приложение представляет собой простой чат-интерфейс:

- сообщения пользователя и собеседника отображаются по-разному;
- список сообщений обновляется через `DiffUtil`;
- поле ввода меняет иконку действия в зависимости от текста;
- при отправке сообщения в список добавляется новый элемент;
- затем имитируется ответ собеседника.

Проект хорошо подходит как учебный пример Android UI, списка сообщений и адаптера с несколькими `ViewHolder`. :contentReference[oaicite:1]{index=1}

## Возможности

- отображение списка сообщений;
- два типа сообщений:
  - свои;
  - чужие;
- разные layout для разных типов элементов;
- обновление списка через `DiffUtil`;
- отправка нового сообщения;
- автоматическая прокрутка к последнему сообщению;
- имитация входящих ответов с задержкой;
- отображение статуса сообщения:
  - `SENT`
  - `DELIVERED`
  - `READ`
- форматирование времени сообщения. :contentReference[oaicite:2]{index=2}

## Структура проекта

- `MainActivity` — основной экран чата и обработка ввода;
- `ChatAdapter` — адаптер списка сообщений;
- `MyChatMessageViewHolder` — отображение своих сообщений;
- `NotMyChatMessageViewHolder` — отображение чужих сообщений;
- `ChatMessage` — модель сообщения;
- `MyOwnView` — заготовка собственного `View`. :contentReference[oaicite:3]{index=3}

## Модель сообщения

Каждое сообщение описывается через `ChatMessage`:

- `id` — уникальный идентификатор;
- `isMine` — принадлежит ли сообщение пользователю;
- `messageText` — текст;
- `status` — статус доставки;
- `date` — дата и время;
- `isChanged` — дополнительный флаг изменения. :contentReference[oaicite:4]{index=4}

## RecyclerView и адаптер

`ChatAdapter` использует два типа элементов:

- `VIEW_TYPE_MY`
- `VIEW_TYPE_NOT_MY`

Тип определяется через поле `isMine`, а затем создаётся соответствующий `ViewHolder`. Это позволяет показывать разные шаблоны сообщений для пользователя и собеседника. :contentReference[oaicite:5]{index=5}

## DiffUtil

Список обновляется через `DiffUtil.calculateDiff(...)`, что даёт:

- более эффективное обновление списка;
- анимации изменений;
- отсутствие полного `notifyDataSetChanged()`.

Сравнение элементов выполняется по:

- `id` — для проверки, это ли тот же объект;
- полному равенству объекта — для проверки содержимого. :contentReference[oaicite:6]{index=6}

## Логика отправки сообщения

В `MainActivity`:

1. пользователь вводит текст;
2. иконка справа переключается:
   - `ic_audio`, если поле пустое;
   - `ic_send`, если есть текст;
3. при нажатии сообщение добавляется в начало списка;
4. RecyclerView прокручивается к позиции `0`;
5. поле ввода очищается. :contentReference[oaicite:7]{index=7}

## Имитация ответа

После отправки используется `postDelayed(...)`:

- через секунду в список добавляются ответы собеседника;
- список снова обновляется;
- экран прокручивается вверх к новому сообщению.

Это делает чат визуально более живым и показывает работу с динамическим обновлением данных. :contentReference[oaicite:8]{index=8}

### MyChatMessageViewHolder
Показывает:

- текст сообщения;
- время;

### NotMyChatMessageViewHolder
Показывает:

Таким образом пользовательские сообщения и ответы визуально разделены. :contentReference[oaicite:9]{index=9}

## Работа со статусами

Для своих сообщений статус отображается через иконки:

- `SENT`
- `DELIVERED`
- `READ`

Это добавляет реалистичность интерфейсу мессенджера. :contentReference[oaicite:10]{index=10}

## Insets и работа с клавиатурой

В `MainActivity` используется `enableEdgeToEdge()` и `WindowInsetsCompat`, чтобы:

- корректно учитывать системные отступы;
- адаптировать нижний паддинг под клавиатуру;
- не перекрывать поле ввода. :contentReference[oaicite:11]{index=11}

## Пользовательский View

`MyOwnView` — заготовка собственного элемента `View` с переопределёнными:

- `onLayout`
- `onMeasure`
- `onDraw`

Сейчас в нём нет логики, но это база для дальнейшей кастомной отрисовки. :contentReference[oaicite:12]{index=12}
